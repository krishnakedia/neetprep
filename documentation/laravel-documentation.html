Laravel Backend Documentation

Welcome to the NEET Prep Laravel Backend Documentation! This guide will help you understand how the API works, how to set it up, and how to extend it.

What is Laravel? Laravel is a PHP web application framework that provides the backend API for NEET Prep. It handles database operations, authentication, and business logic.

📖 Introduction

The NEET Prep backend is a Laravel 11 REST API that provides:

  • Authentication - JWT-based user login/register
  • Subjects & Topics - CRUD operations for study materials
  • Exams & Questions - Exam management and question banks
  • Exam Attempts - Track student progress
  • Admin Features - User and content management
💡 What is an API?

An API (Application Programming Interface) is how the frontend and backend communicate. The frontend sends HTTP requests, and the backend responds with JSON data.

🛠️ Setup & Installation

Prerequisites

  • PHP (version 8.2 or higher) - Download here
  • Composer - PHP package manager - Download here
  • MySQL (version 5.7+) - Or use phpMyAdmin/XAMPP/MAMP

Step 1: Navigate to Backend Folder

cd neetprep-laravel

Step 2: Install Dependencies

composer install

This installs all PHP packages defined in composer.json.

Step 3: Configure Environment

Copy the example environment file:

cp .env.example .env

Or duplicate .env.example and rename it to .env.

Step 4: Create Database

Option A - Using MySQL CLI:

mysql -u root -p
CREATE DATABASE neetprep CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;

Option B - Using phpMyAdmin:

  1. Open phpMyAdmin in your browser
  2. Click "New" to create a database
  3. Name it neetprep
  4. Select utf8mb4_unicode_ci as collation

Step 5: Update .env File

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=neetprep
DB_USERNAME=root
DB_PASSWORD=your_password_here
⚠️ Important: Replace your_password_here with your actual MySQL password. Leave empty if you have no password.

Step 6: Generate Keys

php artisan key:generate

This generates an application key in your .env file.

php artisan jwt:secret

This generates a JWT secret key for authentication tokens.

Step 7: Run Migrations

php artisan migrate

This creates all the database tables.

Step 8: Seed Demo Data

php artisan db:seed

This creates demo users and sample data.

Step 9: Start the Server

php artisan serve

The API will be available at http://localhost:8000

💡 Pro Tip: You can specify a different port:
php artisan serve --port=8080

🏗️ Architecture

Laravel follows the MVC (Model-View-Controller) pattern:

How Requests Flow

  1. Route - User makes HTTP request (e.g., POST /api/auth/login)
  2. Middleware - Request is checked (auth, role, etc.)
  3. Controller - Handles the request logic
  4. Model - Interacts with database
  5. Response - JSON data is sent back

Example Flow: User Login

1. Route: POST /api/auth/login
   ↓
2. Middleware: Check request format
   ↓
3. Controller: AuthController@login
   ↓
4. Model: Validate user credentials against User model
   ↓
5. Response: { success: true, user: {...}, token: "..." }

🎮 Controllers

What is a Controller? A controller handles the logic for processing requests. It's where you decide what happens when an API endpoint is called.

AuthController Example

// app/Http/Controllers/Api/AuthController.php
class AuthController extends Controller
{
    // POST /api/auth/login
    public function login(Request $request)
    {
        // 1. Validate input
        $credentials = $request->validate([
            'email' => 'required|email',
            'password' => 'required'
        ]);

        // 2. Try to authenticate
        if (!$token = auth('api')->attempt($credentials)) {
            return response()->json(['error' => 'Invalid credentials'], 401);
        }

        // 3. Return user and token
        return response()->json([
            'success' => true,
            'user' => auth('api')->user(),
            'token' => $token
        ]);
    }

    // POST /api/auth/logout
    public function logout()
    {
        auth('api')->logout();
        return response()->json(['success' => true]);
    }

    // GET /api/auth/me
    public function me()
    {
        return response()->json([
            'success' => true,
            'user' => auth('api')->user()
        ]);
    }
}

ExamController Example

// app/Http/Controllers/Api/ExamController.php
class ExamController extends Controller
{
    // GET /api/exams (public)
    public function index()
    {
        $exams = Exam::all();

    }

    // GET /api/exams/active (public)
    public function active()
    {
        $exams = Exam::where('isActive', true)->get();
        return response()->json(['success' => true, 'exams' => $exams]);

        $exam = Exam::with(['subject', 'questions'])->find($id);

        }

        return response()->json(['success' => true, 'exam' => $exam]);
    }

    // POST /api/admin/exams (admin only)
    public function store(Request $request)
    {
        // Validate and create exam...
    }
}

🛤️ Routes & Middleware

What are Routes? Routes define which controller method handles each HTTP request.

Route File

// routes/api.php

// Public routes (no auth required)
Route::post('/auth/login', [AuthController::class, 'login']);
Route::post('/auth/register', [AuthController::class, 'register']);

// Public read-only routes
Route::get('/subjects', [SubjectController::class, 'index']);
Route::get('/exams', [ExamController::class, 'index']);

// Protected routes (auth required)
Route::middleware('auth:api')->group(function () {
    Route::post('/auth/logout', [AuthController::class, 'logout']);
    Route::get('/auth/me', [AuthController::class, 'me']);
    
    // User routes
    Route::post('/exam-attempts/start', [ExamAttemptController::class, 'start']);
    Route::post('/exam-attempts/{id}/submit', [ExamAttemptController::class, 'submit']);
    
    // Admin routes
    Route::middleware('role:admin')->group(function () {
        Route::get('/admin/users', [UserController::class, 'index']);
        Route::post('/admin/users', [UserController::class, 'store']);
        Route::put('/admin/users/{id}', [UserController::class, 'update']);
    });
});

Middleware

What is Middleware? Middleware filters requests before they reach the controller.

Auth Middleware

Protects routes - only logged-in users can access:

Route::middleware('auth:api')->group(function () {
    // These routes require valid JWT token
});

Role Middleware

Protects routes - only admins can access:

Route::middleware('role:admin')->group(function () {
    // These routes require admin role
});
💡 Middleware Order: Routes are checked in order. If a route has multiple middleware, they run from left to right. auth:api runs before role:admin.

🔐 Authentication

NEET Prep uses JWT (JSON Web Tokens) for authentication.

How JWT Works

  1. User logs in with email/password
  2. Server validates and generates a token
  3. Token is sent back and stored by frontend
  4. Frontend includes token in all subsequent requests
  5. Server validates token on each request

Token Structure

// Login Response
{
    "success": true,
    "user": {
        "id": "1",
        "name": "Admin User",
        "email": "admin@neetprep.com",
        "role": "admin"
    },
    "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9..."
}

Using the Token

The frontend must include the token in every API request:

// Every request header should include:
Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...
💡 Tip: The Angular app automatically adds this header via an HTTP interceptor. You don't need to do it manually!

Token Expiration

JWT tokens expire after a set time (default: 60 minutes). You can configure this in .env:

JWT_TTL=60  # Token expires in 60 minutes

Custom JWT Guard

The app uses a custom JWT guard configured in:

  • app/Extensions/JWTGuard.php - Custom authentication guard
  • app/Providers/JWTAuthServiceProvider.php - JWT service provider
  • config/auth.php - Auth configuration

❓ Troubleshooting

Common Issues

Error: "No such file or directory" when running artisan

Solution: Make sure you're in the neetprep-laravel directory and run composer install.

Error: "Access denied for user 'root'@'localhost'"

Solution: Check your MySQL credentials in .env file. Make sure the password is correct.

Error: "SQLSTATE[HY000] [2002] No such file or directory"

Solution: Change DB_HOST=127.0.0.1 instead of localhost in .env.

Error: "JWT token is invalid"

Solution:
1. Run php artisan jwt:secret
2. Clear cache: php artisan config:clear

Error: "Table not found"

Solution: Run php artisan migrate to create database tables.

Frontend can't connect to API

Solution:
1. Make sure Laravel is running: php artisan serve
2. Check if it's on port 8000
3. Verify CORS settings allow localhost:4200

Getting Help

  • Run php artisan route:list to see all available routes
  • Check storage/logs/laravel.log for detailed error messages
  • Set APP_DEBUG=true in .env for detailed errors
