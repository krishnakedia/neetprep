Angular Frontend Documentation

Welcome to the NEET Prep Angular Frontend Documentation! This guide will help you understand how the frontend application works, its structure, and how to make changes to it.

What is Angular? Angular is a TypeScript-based web application framework that helps you build interactive web applications. Think of it like building with Lego blocks - you create components (blocks) that fit together to make your app.

📖 Introduction

The NEET Prep frontend is an Angular 21 application that provides:

  • Admin Panel - For managing users, subjects, exams, and questions
  • User Panel - For students to take exams and view results
  • Login System - With role-based access (Admin vs Regular User)
💡 What You'll Learn:
  • How the project is organized
  • How components work
  • How services communicate with the backend
  • How routing protects pages
  • How to add new features

🛠️ Setup & Installation

Prerequisites

Before you start, make sure you have:

  • Node.js (version 18 or higher) - Download here
  • npm (comes with Node.js)
  • Angular CLI - Tool to create Angular projects

Step 1: Install Angular CLI

npm install -g @angular/cli

The -g flag installs it globally so you can use it anywhere.

Step 2: Install Dependencies

npm install

This reads the package.json file and installs all required packages.

Step 3: Start the Development Server

npm start

or

ng serve

The app will be available at http://localhost:4200

💡 Pro Tip: Use npm start for development. It has hot-reload, meaning changes to your code will automatically refresh the browser!

🧩 Components

What is a Component? A component is a reusable piece of UI (User Interface). In Angular, everything you see on screen is a component!

Component Anatomy

Each component typically has:

  • .ts file - The logic/behavior (TypeScript)
  • .html file - The template/markup (HTML)
  • .scss file - The styling (CSS)

How Components Work

import { Component } from '@angular/core';

@Component({
  selector: 'app-my-component',    // HTML tag to use this component
  templateUrl: './my-component.component.html',  // HTML file
  styleUrls: ['./my-component.component.scss']    // Styles
})
export class MyComponent {
  // Your logic here
}

Example: Login Component

// src/app/components/login/login.component.ts
import { Component } from '@angular/core';
import { AuthService } from '../../services/auth.service';

@Component({
  selector: 'app-login',
  templateUrl: './login.component.html',
  styleUrls: ['./login.component.scss']
})
export class LoginComponent {
  email = '';
  password = '';

  constructor(private authService: AuthService) {}

  onLogin() {
    this.authService.login(this.email, this.password)
      .subscribe({
        next: (response) => {
          // Handle successful login
          console.log('Logged in!', response);
        },
        error: (error) => {
          // Handle error
          console.error('Login failed', error);
        }
      });
  }
}
📝 Note: Components use inject() or constructor to get services. The template (.html file) has access to the component's properties and methods.

🔌 Services

What is a Service? A service is a reusable piece of code that provides functionality across your app. Services typically handle:

  • Communicating with the backend API
  • Sharing data between components
  • Business logic

Creating a Service

import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';

@Injectable({
  providedIn: 'root'  // Makes this service available app-wide
})
export class MyService {
  private apiUrl = 'http://127.0.0.1:8000/api';

  constructor(private http: HttpClient) {}

  getData() {
    return this.http.get(`${this.apiUrl}/data`);
  }

  postData(data: any) {
    return this.http.post(`${this.apiUrl}/data`, data);
  }
}

Auth Service (Example)

The AuthService handles all authentication-related tasks:

// Key methods in AuthService:
login(email: string, password: string)  // Login user
logout()                                 // Logout user
getToken()                               // Get JWT token
getCurrentUser()                        // Get logged-in user
isLoggedIn()                            // Check if user is logged in
isAdmin()                               // Check if user is admin
💡 Service Pattern: Services are singletons (one instance) when using providedIn: 'root'. This means the same data is shared across all components.

🧭 Routing

What is Routing? Routing determines which component to show based on the URL. It's like a map that tells Angular which page to display.

Route Configuration

// src/app/app.routes.ts
import { Routes } from '@angular/router';
import { AuthGuard, AdminGuard } from './guards/auth.guard';

export const routes: Routes = [
  { path: '', redirectTo: '/login', pathMatch: 'full' },
  
  // Login page (no protection)
  { 
    path: 'login', 
    loadComponent: () => import('./components/login/login.component')
      .then(m => m.LoginComponent)
  },
  
  // Admin routes (protected by AuthGuard AND AdminGuard)
  { 
    path: 'admin', 
    canActivate: [AuthGuard, AdminGuard],
    loadComponent: () => import('./components/admin/admin-dashboard/admin-dashboard.component')
      .then(m => m.AdminDashboardComponent)
  },
  
  // Regular user routes (protected by AuthGuard only)
  { 
    path: 'user', 
    canActivate: [AuthGuard],
    loadComponent: () => import('./components/user/user-dashboard/user-dashboard.component')
      .then(m => m.UserDashboardComponent)
  },
  
  // Wildcard route (404)
  { path: '**', redirectTo: '/login' }
];

Route Parameters

Some routes need dynamic values (like IDs):

// Route with parameter (lazy loaded)
{ 
  path: 'user/exam/:examId', 
  loadComponent: () => import('./components/user/exam/exam.component')
    .then(m => m.ExamComponent)
}

// Accessing parameter in component
import { ActivatedRoute } from '@angular/router';

constructor(private route: ActivatedRoute) {
  const examId = this.route.snapshot.paramMap.get('examId');
}

🛡️ Guards

What is a Guard? A guard protects routes from unauthorized access. Think of it like a bouncer at a club.

AuthGuard

Ensures only logged-in users can access the route:

// src/app/guards/auth.guard.ts
import { inject } from '@angular/core';
import { Router, CanActivateFn } from '@angular/router';
import { AuthService } from '../services/auth.service';

export const AuthGuard: CanActivateFn = (route, state) => {
  const authService = inject(AuthService);
  const router = inject(Router);

  if (authService.isLoggedIn()) {
    return true;  // Allow access
  }

  router.navigate(['/login']);  // Redirect to login
  return false;  // Deny access
};

AdminGuard

Ensures only admins can access the route:

// AdminGuard is in the same file: src/app/guards/auth.guard.ts
export const AdminGuard: CanActivateFn = (route, state) => {
  const authService = inject(AuthService);
  const router = inject(Router);

  if (authService.isAdmin()) {
    return true;  // Allow access
  }

  router.navigate(['/user']);  // Redirect regular users
  return false;
};
⚠️ Important: Always use both AuthGuard AND AdminGuard for admin routes! AuthGuard checks if logged in, AdminGuard checks if admin.

🔄 Interceptors

What is an Interceptor? An interceptor intercepts all HTTP requests and can modify them. We use it to automatically add the JWT token to every request.

Auth Interceptor

// src/app/interceptors/auth.interceptor.ts
import { inject } from '@angular/core';
import { HttpInterceptorFn } from '@angular/common/http';
import { AuthService } from '../services/auth.service';

export const authInterceptor: HttpInterceptorFn = (req, next) => {
  const authService = inject(AuthService);
  const token = authService.getToken();

  if (token) {
    // Clone the request and add authorization header
    const authReq = req.clone({
      setHeaders: {
        Authorization: `Bearer ${token}`
      }
    });
    return next(authReq);
  }

  return next(req);
};
💡 How it Works: Every time the app makes an HTTP request, the interceptor automatically adds the JWT token. This happens behind the scenes - you don't need to do anything extra!

🔐 Authentication Flow

Here's how the authentication system works:

Login Flow

  1. User enters email and password on the login page
  2. Login component calls AuthService.login()
  3. AuthService sends POST request to /api/auth/login
  4. Backend validates credentials and returns JWT token
  5. AuthService stores token in localStorage
  6. AuthGuard allows navigation to protected routes

Making API Calls

All authenticated API calls automatically include the JWT token:

// No need to manually add headers!
this.http.get('/api/protected-endpoint')
  .subscribe(data => console.log(data));
// The interceptor adds: Authorization: Bearer {token}

Logout Flow

  1. User clicks logout
  2. AuthService calls /api/auth/logout
  3. Token is removed from localStorage
  4. User is redirected to login page

🔨 Build & Deploy

Development Build

npm start

Starts development server at http://localhost:4200 with hot-reload.

Production Build

npm run build

Creates optimized build in dist/neet-prep folder. This output can be deployed to any web server.

Production Build Features:

  • Minified code (smaller file sizes)
  • Uglified JavaScript (harder to read, faster to execute)
  • Tree-shaking (removes unused code)
  • Source maps for debugging (optional)

Watch Mode (Development)

npm run watch

Builds in development mode and watches for file changes.

Run Tests

ng test

Runs unit tests (if any exist).

❓ Troubleshooting

Common Issues

Error: Cannot find module './...'

Solution: Run npm install to ensure all dependencies are installed.

Error: Connection refused to localhost:8000

Solution: Make sure the Laravel backend is running. Start it with cd neetprep-laravel && php artisan serve

Error: JWT Token Expired

Solution: Log out and log back in to get a fresh token. The backend JWT TTL is 60 minutes by default.

Build errors

Solution: Clear the Angular cache and rebuild:
rm -rf node_modules/.cache
npm run build

Getting Help

  • Check the browser console (F12) for error messages
  • Check the Network tab to see API responses
  • Run ng serve --verbose for detailed output
